### Enforce Azure Resource Naming Convention

Implement a standardized naming convention for all Azure resources created through the Terraform UI.

Resource names must be generated dynamically using the following format:
`<short_code>-<project_code>-<environment>-<region>-<instance_number>`

ðŸ”¹ **Example:** For a Key Vault in the `inid` project, `dev` environment, `eastus` region, and instance `01`, the name should be:
`kv-inid-dev-eastus-01`

ðŸ”¹ **For storage accounts** (maximum 24 characters, lowercase only), use the format:
`st<project_code><environment><instance_number>`
**Example:** `stiniddev01`
